Documentation de configuration du Plugin homisarah


date : 15/01/2014
version : 1.0



1. CONFIGURATION


Le fichier homisarah.prop permet de paramétrer les informations nécessaires au bon fonctionnement du plugin.

Il est donc indispensable de renseigner ces 3 éléments :
  • IP

    • Permet d'envoyer des requêtes directement au serveur homidom via le réseau local. Il suffit pour cela d'indiquer l'adresse IP locale du serveur, par défaut localhost
  • port

    • Ce paramètre permet de choisir le port du serveur homidom par défaut 7999.
  • ID

    • Ce paramètre permet de choisir ID du serveur homidom par défaut 123456789.



  • Configuration de homisarah.xml



    • Configuration des composants

    Il faudra modifier les ID de vos composants dans le fichier homisarah.xml, c'est cette partie "fc8a97ae-feaa-4f04-85e8-66721c163dd2" que vous devez
    modifier en fonction de l'ID de vos composants, vous pouvez aussi créer plusieur ligne avec le meme ID ce qui vous permet de faire reconaitre plusieur phrases différente a SARAH

    par exemple

    <item>le salon <tag>out.action.device="fc8a97ae-feaa-4f04-85e8-66721c163dd2"</tag></item>
    <item>les lumiéres du salon<tag>out.action.device="fc8a97ae-feaa-4f04-85e8-66721c163dd2"</tag></item>
    <item>les lampes du salon<tag>out.action.device="fc8a97ae-feaa-4f04-85e8-66721c163dd2"</tag></item>

    Vous permetra de définir plusieurs phrases différente pour le même ID, dans cette exemple vous pouvez dire.

    Sarah, allume le salon
    Sarah, eteint le salon
    Sarah, allume les lampes du salon
    Sarah, eteint les lampes du salon etc...


    Configuration des macros

    Pour les macros et les températures c'est le même principe que pour les composants, il suffit de modifier la partie ID_Macro ou ID_Read, vous trouverez les id des macros dans le fichier
    config.xml de homidom

    Les commande sont

    Sarah, mode éclairage soirée
    Sarah, mode XBMC.
    Sarah, donne moi la température de salon




    Exemple de fichier homisarah.xml



    configuration des messages de confirmation

    SARAH confirme que la commande a bien été éxecutée par different messages que vous pouvez modifier dans le fichier homisarah.xml, voila un exemple, vous pouvez modifier la partie en rouge
    <item>allume<tag>out.action.type="command";out.action.command="on";out.action.ttsAction="j'ai allumer"
    Si vous ne voulez pas de vocalisation supprimer ce qui ce trouve dans la callback comme ceci
    callback({});
